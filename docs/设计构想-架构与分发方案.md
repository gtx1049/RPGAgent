# 设计构想：RPGAgent 架构与分发方案

> 归档时间：2026-03-24
> 状态：构思阶段

---

## 一、核心原则

**引擎和内容分离。**

- `RPGAgent` 是引擎，跟游戏内容（剧本）解耦
- 引擎版本稳定，内容（剧本）频繁更新
- 玩家可以安装不同的剧本包，就像换游戏一样

---

## 二、分层架构

```
┌─────────────────────────────────────────────────────┐
│                    前端层（可替换）                   │
│  [终端CLI]  [Web浏览器]  [独立客户端]  [VSCode插件]  │
├─────────────────────────────────────────────────────┤
│                    引擎核心（稳定）                   │
│  ┌──────────┐  ┌──────────┐  ┌─────────────────┐  │
│  │ AgentScope │  │ 数值系统  │  │  上下文管理器    │  │
│  │   DM Agent │  │(隐藏数值) │  │ (SQLite/Query)  │  │
│  └──────────┘  └──────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────┤
│                    内容层（可替换）                   │
│  ┌──────────────────────────────────────────────┐  │
│  │  games/ {剧本名}/                              │  │
│  │  ├── meta.json   # 剧本元信息                  │  │
│  │  ├── setting.md  # 世界观                     │  │
│  │  ├── characters/ # 人物 JSON                   │  │
│  │  ├── scenes/    # 场景 Markdown               │  │
│  │  ├── systems.yaml # 本剧本专属数值配置         │  │
│  │  └── assets/    # 配图/音效/字体等（可选）    │  │
│  └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

## 三、引擎核心（`rpgagent` Python 包）

发布到 PyPI 或私有源：

```
rpgagent/
├── __init__.py
├── cli.py              # 命令行入口
├── server.py           # FastAPI 服务（供前端调用）
├── engine/
│   ├── game_master.py
│   ├── session.py
│   ├── context_loader.py
│   └── prompt_builder.py
├── systems/            # 数值系统
├── data/              # 数据库层
└── plugins/           # 插件接口（未来扩展）

# 安装：pip install rpgagent
```

## 四、剧本包格式（`gamepkg`）

每个剧本是一个标准化的内容包，可以分发、安装、升级：

```
秦末·大泽乡-1.0.gamepkg     # 单文件分发包（本质是 .zip）
或
rpgagent-game-qinmo/        # pip 安装的剧本包
    ├── __init__.py
    ├── content/            # games/ 的内容
    ├── config.yaml         # 引擎配置（使用哪些隐藏数值等）
    └── requirements.txt    # 依赖（如特定模型配置）
```

**安装方式：**

```bash
# 方式1：pip install
pip install rpgagent-game-qinmo

# 方式2：本地文件
rpgagent install ./秦末·大泽乡-1.0.gamepkg

# 方式3：URL 安装
rpgagent install https://example.com/games/qinmo.gamepkg
```

剧本安装后放到 `~/.config/rpgagent/games/` 目录。

## 五、前端接口（Engine API）

引擎通过 HTTP/WebSocket 对外提供接口，任何前端都可以接入：

### REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/games` | GET | 列出已安装剧本 |
| `/games/{id}/start` | POST | 开始新游戏 |
| `/games/{id}/scenes/{scene_id}` | GET | 获取场景内容 |
| `/action` | POST | 玩家行动 → 返回叙事 |
| `/status` | GET | 当前玩家状态 |
| `/npcs` | GET | 当前 NPC 列表 |
| `/history` | GET | 叙事历史 |
| `/log` | GET | 冒险日志列表 |
| `/saves` | GET/POST/DELETE | 存档管理 |

### WebSocket（实时叙事流）

```
WS /game/{session_id}/stream

# 服务器推送（叙事分段下发）：
{ "type": "narrative", "content": "你走进...", "done": false }
{ "type": "narrative", "content": "雨在下。", "done": false }
{ "type": "options", "options": [...] }
{ "type": "done" }

# 客户端发送：
{ "action": "输入框内容" }
```

## 六、CLI 工具

安装引擎后全局可用 `rpg` 命令：

```bash
rpg list              # 列出已安装剧本
rpg install ./xxx.gamepkg   # 安装剧本
rpg start 秦末        # 开始游戏（默认终端界面）
rpg start 秦末 --web  # 启动 Web 服务器 + 打开浏览器
rpg serve             # 仅启动 API 服务器（供远程前端）
rpg init              # 创建新剧本项目脚手架
```

## 七、打包发行方案

### 方案 A：纯 PyPI（适合开发者/Gamer）

```bash
# 用户安装
pip install rpgagent
pip install rpgagent-game-qinmo

# 启动
rpg start 秦末
```

### 方案 B：独立可执行文件（适合非技术玩家）

用 PyInstaller 打包：

```
dist/
├── rpgagent.exe       # Windows 启动器
├── rpgagent_mac       # macOS 启动器
└── resources/
    ├── games/         # 内置示例剧本
    └── assets/        # 默认素材
```

用户下载 exe → 双击运行 → 浏览器自动打开 `localhost:7860`

### 方案 C：Docker（适合服务器部署）

```bash
docker run -p 7860:7860 \
  -v ~/.config/rpgagent:/root/.config/rpgagent \
  rpgagent/agent:latest
```

## 八、剧本分发平台

未来可以搭建简单的剧本市场：

```
rpgagent.market/
├── 剧本列表（浏览/搜索）
├── 评分 + 评论
├── 下载计数
└── 提交规范（gamepkg 校验）
```

每个剧本包需要包含：
- `meta.json`（格式校验）
- `manifest.yaml`（引擎版本要求/标签/作者）
- 内容文件完整性校验（SHA256）

## 九、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | 轻量、自动 OpenAPI docs |
| WebSocket | FastAPI 内置 | 与后端同框架 |
| 前端（默认） | 纯 HTML/JS | 零依赖，可嵌入 |
| 数据库 | SQLite | 引擎内置，用户无需装数据库 |
| 发行格式 | gamepkg（zip重命名）| 简单，自包含 |
| 引擎打包 | PyInstaller | 独立可执行文件 |
| 服务器部署 | Docker | 一键部署 |
