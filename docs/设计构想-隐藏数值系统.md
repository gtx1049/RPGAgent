## 核心洞察

道德债务只是一个具体实例。真正的抽象是：

**玩家行为在背后积累某种"账"，这个账随着时间影响玩家能看到/想到/愿意做的事。**

---

## 1. HiddenValue — 通用原子单位

```python
class HiddenValue:
    id: str                     # "moral" / "sanity" / "development"
    name: str                   # 展示名："道德债务" / "理智" / "发展度"
    description: str            # 解释这个值代表什么

    level: int                  # 当前等级（阈值决定档位）
    records: List[Record]       # 积累记录（来源/场景/描述/时间）

    # 等级 → 叙事体验映射
    level_effects: Dict[int, LevelEffect]
    # 等级 → 可用选项锁定
    level_gates: Dict[int, List[str]]
    # 特殊触发：当跨过某阈值时触发的事件
    threshold_events: Dict[int, str]  # level → 场景ID
```

## 2. HiddenValueSystem — 多值并行，互相影响

```python
class HiddenValueSystem:
    values: Dict[str, HiddenValue]

    # 一次行为可能同时触发多个隐藏数值
    # 比如你目睹屠杀：
    #   moral_debt +5
    #   sanity     -3
    #   development +2（你学会了一些"生存技巧"）

    def record_action(self, action: str, scene: str) -> Dict[str, int]:
        """返回各隐藏值的变化量"""
```

## 3. 核心设计原则：玩家看不到数值，看到的是体验

```
sanity（理智值）举例：

80%+：世界是完整的，叙事语言正常
50-80%：边缘开始模糊。NPC的话有时候会有第二层意思
30-50%：你开始看到不存在的东西。叙事视角偶尔跳脱，句子变短
         某些选项不再出现——你"想不起来"还有那个选择
<30%：彻底碎片化叙事。玩家只能选择"攻击/逃跑/僵住"
```
