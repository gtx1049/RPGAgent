# 设计构想：隐藏数值系统作为 AgentScope ToolCall

> 归档时间：2026-03-24
> 状态：**已实现**（core/gms_tools.py + core/game_master.py）
> 实现时间：2026-03-26

---

## 核心思路

将隐藏数值系统实现为 AgentScope 的 `ToolCall` 能力，而不是被动响应 Prompt 注入。

这样 LLM 可以**主动调用**数值系统，而不是等叙事结束后由 `_execute_command` 事后更新。

---

## 为什么用 ToolCall？

**传统方式（Prompt 注入）：**
```
每次叙事 → 解析 GM_COMMAND → 事后更新数值
问题：LLM 不知道数值实时状态，只能"猜"或"假设"
```

**ToolCall 方式（主动查询/更新）：**
```
LLM 在叙事过程中 → 主动调用 tool("check_hidden_value", id="moral_debt")
                              → 主动调用 tool("record_action", id="moral", delta=5)
LLM 在生成选项时 → 调用 tool("get_locked_options", hidden_value_ids=["moral"])
                              → 根据结果决定哪些选项可用
```
优点：
- LLM 能实时知道当前状态，不依赖猜测
- 数值更新和叙事在同一轮对话内完成，一致性强
- 可以实现"条件分支"：如果 moral >= 11，生成特定选项

## AgentScope Tool 定义

```python
from agentscope import tool

@tool(name="check_hidden_value")
def check_hidden_value(hidden_value_id: str) -> dict:
    """查询指定隐藏数值的当前状态"""
    # 返回当前 level/threshold/locked_options/effects
    ...

@tool(name="get_all_hidden_values")
def get_all_hidden_values() -> dict:
    """获取所有隐藏数值的快照"""
    ...

@tool(name="record_player_action")
def record_player_action(
    hidden_value_id: str,
    delta: int,
    source: str,
    scene_id: str,
    player_action: str,
) -> dict:
    """记录玩家行为导致的隐藏数值变化"""
    ...

@tool(name="get_locked_options")
def get_locked_options(hidden_value_ids: list[str]) -> list[str]:
    """查询当前被锁定的选项类型"""
    ...

@tool(name="check_threshold_trigger")
def check_threshold_trigger(hidden_value_id: str) -> str | None:
    """检查是否跨过阈值，返回触发的场景ID或None"""
    ...
```

## LLM 调用流程示例

```
用户输入: "我选择袖手旁观，看着士兵行凶"

LLM推理:
  1. 调用 tool("record_player_action",
        hidden_value_id="moral_debt",
        delta=5,
        source="目睹暴行",
        scene_id="scene_01",
        player_action="袖手旁观")
     → 返回 {"new_value": 15, "level": "微债"}

  2. 调用 tool("get_locked_options", hidden_value_ids=["moral"])
     → 返回 ["道德洁癖选项"]  （当前档位未锁定任何普通选项）

  3. 调用 tool("check_threshold_trigger", hidden_value_id="moral_debt")
     → 返回 None  （未跨阈值）

  4. 根据以上结果生成叙事 + 可用选项
```

---

## ToolCall 触发时机

LLM 在两种时刻主动调用工具：

| 时机 | 调用的 Tool | 目的 |
|------|-------------|------|
| 玩家行动后 | `record_player_action` | 更新数值 |
| 生成选项前 | `get_locked_options` | 过滤不可用选项 |
| 叙事分支判断前 | `check_hidden_value` | 判断条件分支 |
| 场景切换时 | `check_threshold_trigger` | 检查延迟事件触发 |

## 冒险日志生成

### 触发时机

当玩家完成一幕（Act）冒险时，调用日志生成：

```python
@tool(name="generate_adventure_log")
def generate_adventure_log(
    act_id: str,
    act_title: str,
    turns: int,
    events: list[dict],
    hidden_value_summary: dict,
    final_stats: dict,
) -> str:
    """
    生成冒险日志（Markdown格式）。
    供玩家回顾本次冒险经历。
    """
```

### 日志内容结构

```markdown
# 冒险日志 · 第一幕 · 大泽乡

**冒险时长**：第 1 回合 — 第 23 回合
**游玩日期**：2026-03-24

## 关键事件

- 🔸 雨夜决策：你选择在大雨中等待，目睹了尉官虐待同伴
- 🔸 狐鸣篝火：你保持沉默，未出声响应
- 🔸 杀尉之夜：你参与了杀尉行动（道德债务 +6）

## 角色状态

| 数值 | 变化 |
|------|------|
| 道德债务 | 0 → 11（+11）|
| 理智 | 100 → 95（-5）|
| 体力 | 100 → 72（-28）|

## 关系变化

- 👤 陈胜：陌生 → 熟悉（+5）
- 👤 吴广：陌生 → 普通（+3）
- 👤 士兵甲：陌生 → 冷淡（-3）

## 债务记录

- 目睹暴行 +5（第一夜，尉官虐待同伴）
- 沉默旁观 +3（雨中抉择）
- 参与围观 +3（杀尉之夜）

## 本幕印象

> "你没有说话。你只是站在那里，看着火把照亮那些人的脸。
> 你知道自己做了什么——也知道自己没做什么。"

---

**解锁成就**：
- 🏅 大泽乡幸存者

```

### 日志存储

- 保存到 `games/{game_id}/logs/act_{n}_{timestamp}.md`
- 也可作为 context 摘要，在新一幕开始时注入，帮助 LLM 回忆前情
